# Contributing Guidelines

Thank you for your interest in contributing to OptiSchedule Pro! We welcome contributions from everyone.

## Getting Started

1.  **Fork the repository**: Click the "Fork" button in the top right corner of the repository page.
2.  **Clone your fork**: Clone the repository to your local machine.
3.  **Install dependencies**: Run `npm install` to install the necessary dependencies.
4.  **Create a branch**: Create a new branch for your feature or bug fix.

## Development Workflow

1.  **Make changes**: Implement your changes in the new branch.
2.  **Test your changes**: Ensure that your changes work as expected and do not introduce any new bugs.
3.  **Commit your changes**: Commit your changes with a clear and descriptive commit message.
4.  **Push your changes**: Push your changes to your fork.
5.  **Create a Pull Request**: Submit a Pull Request to the main repository.

## Security Guidelines

*   **Do not commit secrets**: Ensure that no API keys, passwords, or other sensitive information are committed to the repository. Use environment variables instead.
*   **Report vulnerabilities**: If you discover a security vulnerability, please follow the instructions in [SECURITY.md](SECURITY.md) to report it responsibly.
*   **Code Review**: All contributions will be reviewed by a team member before being merged.
*   **Dependencies**: Keep dependencies up to date and check for known vulnerabilities.

## Code Style

*   Follow the existing code style and formatting conventions.
*   Use meaningful variable and function names.
*   Write clear and concise comments where necessary.
