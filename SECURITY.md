## Reporting a Vulnerability

We take the security of our software seriously. If you suspect that you have found a security vulnerability in this project, please report it to us immediately.

### How to Report

1.  **Do not** open a GitHub issue.
2.  Please send an email to security@your-project.com (replace with actual security contact) with a full description of the vulnerability.
3.  Please include the steps required to reproduce the issue.
4.  We will acknowledge your report within 48 hours.

### Response Timeline

*   We will investigate the issue and verify its validity within 5 business days.
*   If the issue is valid, we will work on a fix and release it as soon as possible.
*   We will notify you when the fix is released.

### Disclosure Policy

*   We request that you do not publicly disclose the vulnerability until we have had a chance to fix it.
*   We will credit you for your discovery in our release notes (unless you prefer to remain anonymous).

## Security Best Practices for Contributors

*   Ensure that no secrets (API keys, passwords, etc.) are committed to the repository.
*   Use environment variables for sensitive configuration.
*   Keep dependencies up to date to avoid known vulnerabilities.
*   Follow the principle of least privilege when configuring access controls.
